# MAX30101 心率 / 血氧模式寄存器配置说明

> 基于 MAX30101 数据手册 (Rev 2) 整理, 供项目开发时快速查阅和调参使用。

---

## 1. 芯片概述

| 项目 | 说明 |
|------|------|
| 型号 | MAX30101 (模组集成版: MAX30101EFD+) |
| 功能 | 集成式脉搏血氧仪和心率监测传感器 |
| 内部 LED | Red (660nm), IR (880nm), Green (530nm), 可选 Pilot LED |
| 接口 | I2C, 7-bit 地址 = 0x57, 最高 400kHz |
| ADC 分辨率 | 18-bit (由 LED 脉宽寄存器决定, 15/16/17/18 bit) |
| FIFO 深度 | 32 个采样点 |
| 模拟供电 | 1.7V ~ 2.0V |
| LED 供电 | 3.1V ~ 5.25V |
| Part ID | 寄存器 0xFF = 0x15 |
| Revision ID | 寄存器 0xFE |

### 三种工作模式

| MODE[2:0] | 模式 | 激活的 LED | FIFO 每采样字节数 |
|-----------|------|-----------|------------------|
| 0x02 | 心率 (Heart Rate) | LED1 (Red) | 3 字节 |
| 0x03 | 血氧 (SpO2) | LED1 (Red) + LED2 (IR) | 6 字节 |
| 0x07 | 多 LED (Multi-LED) | 由 Slot 寄存器配置 | 3 x 活跃 Slot 数 |

## 2. 心率模式配置 (Heart Rate Mode)

心率模式只使用 **Red LED**, 适用于手腕/指尖心率检测。

### 2.1 典型配置流程 (寄存器写入顺序)

```
1. 复位     -> 写 0x09 = 0x40  (RESET=1)
2. 中断使能 -> 写 0x02 = 0xC0  (A_FULL_EN + PPG_RDY_EN)
3. FIFO 配置 -> 写 0x08 = 0x4F (SMP_AVE=4x, ROLLOVER=1, A_FULL=15)
4. SpO2 配置 -> 写 0x0A = 0x27 (ADC=4096nA, Rate=100Hz, PW=411us)
5. LED 电流  -> 写 0x0C = 0x24 (Red LED = 7.2mA)
6. 启动模式 -> 写 0x09 = 0x02 (MODE=Heart Rate)
```

### 2.2 关键寄存器详解

#### 0x09 - 模式配置寄存器 (Mode Configuration)

| Bit | 名称 | 说明 |
|-----|------|------|
| 7 | SHDN | 1 = 软件关断 (省电), 0 = 正常工作 |
| 6 | RESET | 写 1 触发一次完整复位, 自动清零 |
| 5:3 | - | 保留, 写 0 |
| 2:0 | MODE[2:0] | 010 = 心率模式 |

**心率模式**: 写入 `0x02` (正常工作) 或 `0x82` (关断状态)。

#### 0x0A - SpO2 配置寄存器 (心率模式同样使用此寄存器)

此寄存器控制 ADC 精度、采样率和 LED 脉宽, **两种模式共用**。

| Bit | 名称 | 说明 |
|-----|------|------|
| 7:6 | SPO2_ADC_RGE[1:0] | ADC 满量程范围 |
| 5:3 | SPO2_SR[2:0] | 采样率选择 |
| 2 | - | 保留, 写 0 |
| 1:0 | LED_PW[1:0] | LED 脉冲宽度 / ADC 分辨率 |

**ADC 满量程范围 (SPO2_ADC_RGE)**:

| 值 | 满量程 (nA) | LSB (pA) | 适用场景 |
|----|------------|----------|---------|
| 00 | 2048 | 7.81 | 低功耗, 信号较强 |
| 01 | 4096 | 15.63 | 通用默认 |
| 10 | 8192 | 31.25 | 信号较暗/皮肤较深 |
| 11 | 16384 | 62.50 | 极弱信号, 最大动态范围 |

**采样率 (SPO2_SR)**:

| 值 | 采样率 (sps) | 备注 |
|----|-------------|------|
| 000 | 50 | 极低功耗 |
| 001 | 100 | 心率模式常用 |
| 010 | 200 | 高精度心率 |
| 011 | 400 | 运动/快速变化 |
| 100 | 800 | 研究用途 |
| 101 | 1000 | 高速采集 |
| 110 | 1600 | 极速 |
| 111 | 3200 | 最大速率 |

> **注意**: 有效采样率 = 设定值 / 平均次数 (SMP_AVE)。例如设定 400sps + 平均4次 = 实际 100sps 输出。

**LED 脉冲宽度 (LED_PW)**:

| 值 | 脉宽 (us) | ADC 位数 | 备注 |
|----|----------|---------|------|
| 00 | 69 | 15-bit | 低功耗, 精度最低 |
| 01 | 118 | 16-bit |  |
| 10 | 215 | 17-bit |  |
| 11 | 411 | 18-bit | 精度最高, 推荐 |

> **脉宽约束**: 脉宽决定 ADC 积分时间, 脉宽越长, 噪声越低, 但功耗越大。同时脉宽 + 采样率存在约束: `脉宽 x 活跃LED数 x 采样率 < 100%`。

#### 0x0C - LED1 脉冲幅值 (Red LED)

| 范围 | 步进 | 默认值 | 说明 |
|------|------|--------|------|
| 0x00 ~ 0xFF (0 ~ 255) | 约 0.2mA/step | 0x24 (7.2mA) | 0x00 = 关闭 LED |

**心率模式 LED 电流选择指南**:

| 场景 | 推荐值 | 电流 | 说明 |
|------|--------|------|------|
| 手腕 (反射式) | 0x24 ~ 0x3C | 7.2 ~ 12.0 mA | 常用范围 |
| 指尖 (透射式) | 0x0F ~ 0x24 | 3.0 ~ 7.2 mA | 信号较强, 无需太高 |
| 手指紧按 | 0x0A ~ 0x18 | 2.0 ~ 4.8 mA | 信号很强 |
| 深肤色 / 厚组织 | 0x3C ~ 0x7F | 12.0 ~ 25.5 mA | 需更大驱动 |

#### 0x08 - FIFO 配置寄存器

| Bit | 名称 | 说明 |
|-----|------|------|
| 7:5 | SMP_AVE[2:0] | 采样平均次数 |
| 4 | FIFO_ROLLOVER_EN | FIFO 溢出时是否循环覆盖 |
| 3:0 | FIFO_A_FULL[3:0] | 几乎满阈值 |

**采样平均 (SMP_AVE)**:

| 值 | 平均次数 | 说明 |
|----|---------|------|
| 000 | 1 (无平均) | 保持原始采样率 |
| 001 | 2 | 轻度降噪 |
| 010 | 4 | 推荐 |
| 011 | 8 | 较强降噪 |
| 100 | 16 | 强降噪 |
| 101 | 32 | 最强降噪 |

**FIFO 几乎满阈值 (A_FULL)**:
- 中断触发条件: FIFO 中未读样本数 = `32 - A_FULL[3:0]`
- 例如 A_FULL = 0x0F (15) -> 当 FIFO 有 32-15=17 个样本时触发中断
- 例如 A_FULL = 0x00 -> 当 FIFO 有 32 个样本 (全满) 时触发中断

## 3. 血氧模式配置 (SpO2 Mode)

血氧模式同时使用 **Red LED + IR LED**, 通过双波长吸光度比值计算 SpO2。

### 3.1 典型配置流程

```
1. 复位     -> 写 0x09 = 0x40  (RESET=1)
2. 中断使能 -> 写 0x02 = 0xC0  (A_FULL_EN + PPG_RDY_EN)
3. FIFO 配置 -> 写 0x08 = 0x4F (SMP_AVE=4x, ROLLOVER=1, A_FULL=15)
4. SpO2 配置 -> 写 0x0A = 0x27 (ADC=4096nA, Rate=100Hz, PW=411us)
5. Red LED  -> 写 0x0C = 0x24 (Red LED = 7.2mA)
6. IR LED   -> 写 0x0D = 0x24 (IR LED = 7.2mA)
7. 启动模式 -> 写 0x09 = 0x03 (MODE=SpO2)
```

### 3.2 与心率模式的差异

| 项目 | 心率模式 | 血氧模式 |
|------|---------|---------|
| MODE 值 | 0x02 | 0x03 |
| 活跃 LED | 仅 Red (LED1) | Red (LED1) + IR (LED2) |
| LED2_PA (0x0D) | 不需要 | 必须配置 |
| FIFO 每采样字节数 | 3 | 6 (Red 3字节 + IR 3字节) |
| FIFO 有效容量 | 32 采样 | 32 采样 (但占 192 字节) |
| 数据读取顺序 | 仅 Red | 先 Red 后 IR |
| 采样率/ADC 范围 | 使用 0x0A | 使用 0x0A (相同) |
| 算法需求 | 单通道 PPG | R/IR 比值计算 SpO2 |

### 3.3 IR LED 电流 (0x0D)

格式与 Red LED 寄存器 (0x0C) 完全相同。

**血氧模式 LED 电流调参要点**:
- Red 和 IR 的电流通常需要独立调整, 使两通道 PPG 信号幅值大致相当
- AC 分量太弱 -> 增大电流; DC 分量饱和 (接近 ADC 满量程) -> 减小电流
- 目标: 使 AC/DC 比值在 0.5% ~ 5% 范围, 信号不过载也不太弱
- 典型值: Red 0x24 (7.2mA), IR 0x24 (7.2mA), 实际需要根据个体差异微调

## 4. 寄存器速查表

### 4.1 完整寄存器映射

| 地址 | 名称 | 默认值 | 说明 |
|------|------|--------|------|
| 0x00 | INTR_STATUS_1 | 0x00 | 中断状态 1 (只读) |
| 0x01 | INTR_STATUS_2 | 0x00 | 中断状态 2 (只读) |
| 0x02 | INTR_ENABLE_1 | 0x00 | 中断使能 1 |
| 0x03 | INTR_ENABLE_2 | 0x00 | 中断使能 2 |
| 0x04 | FIFO_WR_PTR | 0x00 | FIFO 写指针 [4:0] |
| 0x05 | OVF_COUNTER | 0x00 | FIFO 溢出计数 [4:0] |
| 0x06 | FIFO_RD_PTR | 0x00 | FIFO 读指针 [4:0] |
| 0x07 | FIFO_DATA | 0x00 | FIFO 数据 (只读, 3字节/通道) |
| 0x08 | FIFO_CONFIG | 0x4F | FIFO 配置 |
| 0x09 | MODE_CONFIG | 0x40 | 模式配置 |
| 0x0A | SPO2_CONFIG | 0x27 | SpO2/ADC/采样率配置 |
| 0x0B | - | - | 保留 |
| 0x0C | LED1_PA | 0x24 | Red LED 脉冲电流 |
| 0x0D | LED2_PA | 0x24 | IR LED 脉冲电流 |
| 0x0E | LED3_PA | 0x24 | Green LED 脉冲电流 |
| 0x0F | PILOT_PA | 0x00 | Pilot LED 脉冲电流 |
| 0x10 | PROX_INT_THRESH | 0x00 | 接近中断阈值 |
| 0x11 | MULTI_LED_CTRL1 | 0x00 | 多 LED 时序控制 (Slot 1-2) |
| 0x12 | MULTI_LED_CTRL2 | 0x00 | 多 LED 时序控制 (Slot 3-4) |
| 0x13~0x1E | - | - | 保留 |
| 0x1F | TEMP_INTR | 0x00 | 芯片温度整数部分 (只读) |
| 0x20 | TEMP_FRAC | 0x00 | 芯片温度小数部分 (只读) |
| 0x21 | TEMP_CONFIG | 0x00 | 温度转换控制 |
| 0xFE | REVISION_ID | - | 版本 ID (只读) |
| 0xFF | PART_ID | 0x15 | 零件 ID (只读) |

### 4.2 中断寄存器位定义

**INTR_STATUS_1 / INTR_ENABLE_1 (0x00, 0x02)**:

| Bit | 名称 | 说明 |
|-----|------|------|
| 7 | A_FULL | FIFO 几乎满中断 |
| 6 | PPG_RDY | 新 PPG 数据就绪 |
| 5 | ALC_OVF | 环境光抵消溢出 |
| 4 | PROX_INT | 接近阈值触发 |
| 3:0 | - | 保留 |

**INTR_STATUS_2 / INTR_ENABLE_2 (0x01, 0x03)**:

| Bit | 名称 | 说明 |
|-----|------|------|
| 1 | DIE_TEMP_RDY | 芯片温度转换完成 |
| 其他 | - | 保留 |

> 读 INTR_STATUS 会自动清除中断标志。

### 4.3 温度寄存器

| 寄存器 | 说明 |
|--------|------|
| 0x1F | TEMP_INT: 温度整数部分 (二进制补码, 8-bit, 单位 degC) |
| 0x20 | TEMP_FRAC: 温度小数部分 (bit[3] = 0.0625 degC, bit[2] = 0.125, ...) |
| 0x21 | bit[0] = TEMP_EN: 写 1 启动一次温度转换, 完成后自动清零 |

温度换算公式: `T(degC) = TEMP_INT + TEMP_FRAC * 0.0625`

## 5. Multi-LED 模式 (Slot 配置)

当 MODE = 0x07 时, 需要通过 Slot 寄存器配置每个时序槽位使用哪个 LED。

#### 0x11 - MULTI_LED_CTRL1

| Bit | 名称 | 说明 |
|-----|------|------|
| 6:4 | SLOT1[2:0] | 时序槽 1 的 LED 选择 |
| 2:0 | SLOT2[2:0] | 时序槽 2 的 LED 选择 |

#### 0x12 - MULTI_LED_CTRL2

| Bit | 名称 | 说明 |
|-----|------|------|
| 6:4 | SLOT3[2:0] | 时序槽 3 的 LED 选择 |
| 2:0 | SLOT4[2:0] | 时序槽 4 的 LED 选择 |

**Slot LED 映射**:

| 值 | LED |
|----|-----|
| 0 | 无 (关闭) |
| 1 | LED1 (Red 660nm) |
| 2 | LED2 (IR 880nm) |
| 3 | LED3 (Green 530nm) |
| 4 | Pilot LED |

> 示例: 同时使用 Red + IR + Green = Slot1=Red, Slot2=IR, Slot3=Green -> 0x12 = 0x30, 0x11 = 0x12。

## 6. FIFO 数据读取

### 6.1 数据格式

每个 LED 通道产生 **3 字节** (18-bit) 数据:

```
Byte 0: [D17, D16, D15, D14, D13, D12, D11, D10]  (高字节)
Byte 1: [D9,  D8,  D7,  D6,  D5,  D4,  D3,  D2]   (中字节)
Byte 2: [D1,  D0,  0,   0,   0,   0,   0,   0]     (低字节, 有效位取决于 ADC 位数)
```

### 6.2 读取方式

1. 读取 FIFO_WR_PTR (0x04) 和 FIFO_RD_PTR (0x06)
2. 计算可用样本数: `NUM_AVAILABLE = WR_PTR - RD_PTR (mod 32)`
3. 从 FIFO_DATA (0x07) 连续读取, 每个样本读 3*N 字节 (N = 活跃 LED 通道数)
4. 芯片自动递增 FIFO 读指针

### 6.3 心率模式读取示例

```c
// 心率模式: 每样本 3 字节 (仅 Red)
uint8_t buf[3 * 32]; // 最多读 32 个样本
max30101_read_fifo(buf, num_samples * 3);

for (int i = 0; i < num_samples; i++) {
    uint32_t red = ((uint32_t)buf[i*3] << 16) |
                   ((uint32_t)buf[i*3+1] << 8) |
                   buf[i*3+2];
    red &= 0x03FFFF;  // 18-bit mask
}
```

### 6.4 血氧模式读取示例

```c
// 血氧模式: 每样本 6 字节 (Red + IR)
uint8_t buf[6 * 32];
max30101_read_fifo(buf, num_samples * 6);

for (int i = 0; i < num_samples; i++) {
    uint32_t red = ((uint32_t)buf[i*6] << 16) |
                   ((uint32_t)buf[i*6+1] << 8) |
                   buf[i*6+2];
    red &= 0x03FFFF;

    uint32_t ir = ((uint32_t)buf[i*6+3] << 16) |
                  ((uint32_t)buf[i*6+4] << 8) |
                  buf[i*6+5];
    ir &= 0x03FFFF;
}
```

## 7. 接近检测 (Proximity Detection)

用于检测手指/皮肤是否贴近传感器, 可作为自动开关采样的触发条件。

| 寄存器 | 地址 | 说明 |
|--------|------|------|
| PROX_INT_THRESH | 0x10 | 阈值 (8-bit), 当 IR 检测值超过此值触发 PROX_INT 中断 |
| PILOT_PA | 0x0F | 接近模式使用的 Pilot LED 电流 (仅在接近模式下) |

- 在心率模式下, 接近检测使用 LED1 (Red)
- 在血氧模式下, 接近检测使用 LED1 (Red) + LED2 (IR)
- 使能接近中断: INTR_ENABLE_1 的 PROX_INT 位 (bit 4) = 1

## 8. 功耗与调参参考

### 8.1 功耗影响因素

| 参数 | 增大 -> 功耗 | 增大 -> 信号质量 |
|------|------------|----------------|
| LED 电流 (0x0C/0x0D) | 升高 | 改善 (更高 SNR) |
| LED 脉宽 (0x0A[1:0]) | 升高 | 改善 (更高 ADC 位数) |
| 采样率 (0x0A[5:3]) | 升高 | 改善 (时间分辨率) |
| 平均次数 (0x08[7:5]) | 升高 | 改善 (降噪) |
| ADC 范围 (0x0A[7:6]) | 无影响 | 动态范围增大但 LSB 增大 |

### 8.2 参数组合约束

```
LED 脉宽 x 活跃 LED 通道数 x 采样率 <= 100%
```

常见合规组合:

| 脉宽 | 采样率 | HR模式 (1ch) | SpO2模式 (2ch) | Multi-LED (3ch) |
|------|--------|-------------|---------------|----------------|
| 411us | 100 Hz | 合规 (4.1%) | 合规 (8.2%) | 合规 (12.3%) |
| 411us | 400 Hz | 合规 (16.4%) | 合规 (32.8%) | 合规 (49.2%) |
| 411us | 800 Hz | 合规 (32.8%) | 合规 (65.6%) | 超限 (98.4%) |
| 411us | 1000 Hz | 合规 (41.1%) | 超限 (82.2%) | 超限 |
| 215us | 1000 Hz | 合规 (21.5%) | 合规 (43.0%) | 合规 (64.5%) |
| 69us | 3200 Hz | 合规 (22.1%) | 合规 (44.2%) | 合规 (66.2%) |

> **超限警告**: 超过 100% 时序时, 芯片行为未定义, 数据可能损坏。

### 8.3 推荐配置组合

**低功耗心率**:
```
0x0A = 0x24 (ADC=4096, Rate=50Hz, PW=215us)
0x0C = 0x18 (Red=4.8mA)
0x08 = 0x4F  (Avg=4x -> 有效 12.5sps)
```

**标准心率** (本项目可能适用):
```
0x0A = 0x27 (ADC=4096, Rate=100Hz, PW=411us)
0x0C = 0x24 (Red=7.2mA)
0x08 = 0x4F  (Avg=4x -> 有效 25sps)
```

**高精度心率**:
```
0x0A = 0x2F (ADC=8192, Rate=400Hz, PW=411us)
0x0C = 0x24 (Red=7.2mA)
0x08 = 0x4F  (Avg=4x -> 有效 100sps)
```

**标准血氧**:
```
0x0A = 0x27 (ADC=4096, Rate=100Hz, PW=411us)
0x0C = 0x24 (Red=7.2mA)
0x0D = 0x24 (IR=7.2mA)
0x08 = 0x4F  (Avg=4x -> 有效 25sps)
```

**高精度血氧**:
```
0x0A = 0x2F (ADC=8192, Rate=400Hz, PW=411us)
0x0C = 0x24 (Red=7.2mA)
0x0D = 0x24 (IR=7.2mA)
0x08 = 0x4F  (Avg=4x -> 有效 100sps)
```

## 9. 常见调参问题与排查

| 现象 | 可能原因 | 调整方法 |
|------|---------|---------|
| 信号全是 0 | LED 未启动或电流为 0 | 检查 MODE 配置和 LED_PA 值 |
| 信号饱和 (固定最大值) | ADC 范围太小或 LED 电流太大 | 降低 LED_PA 或增大 ADC_RGE |
| 信号噪声大 | ADC 位数太低或无平均 | 增大 LED_PW (选 18-bit) + 启用 SMP_AVE |
| 心率不准 | 采样率太低 | 提高采样率到至少 100Hz (有效) |
| FIFO 溢出频繁 | 读取速度跟不上采样率 | 降低采样率或启用 FIFO_ROLLOVER |
| SpO2 值异常 | Red/IR 电流不匹配 | 调整使两通道 AC 幅值接近 |
| 功耗过高 | 采样率/电流过高 | 降低采样率, 减小 LED 电流 |
| 时序违规 | 脉宽 x LED数 x 采样率 > 100% | 减小脉宽或采样率 |

## 10. I2C 通信注意事项

- **设备地址**: 7-bit = 0x57, 写操作 = 0xAE, 读操作 = 0xAF
- **写寄存器**: START -> 0xAE -> REG_ADDR -> DATA -> STOP
- **读寄存器**: START -> 0xAE -> REG_ADDR -> RE-START -> 0xAF -> DATA -> STOP
- **FIFO 读取**: 支持连续读, 地址自动保持在 0x07 不递增
- **上电默认**: 所有配置寄存器有默认值, 建议每次使用前完整初始化
